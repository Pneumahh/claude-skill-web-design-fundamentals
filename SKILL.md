# Web Design Principles And Execution

## Core Rules

- Use a formal type scale system (e.g., Major Third) to determine font sizes intentionally rather than eyeballing them.
- Implement a column grid system: 12 columns for desktop, 8 for tablet, and 4 for mobile for flexible structure.
- Apply an 8-point grid system where all spacing and margins are multiples of 8 to ensure rhythmic consistency.
- Follow the 60-30-10 rule for color palettes: 60% neutrals, 30% secondary, and 10% accent colors.
- Prioritize contrast ratios of at least 3:1 for large text and 4.5:1 for small text to ensure accessibility.
- Design every page around a single clear goal with strategic CTAs placed in the hero, nav, and every 2-3 scrolls.
- Use opacity variations of a primary color instead of adding entirely new colors to your palette.

## When to Apply

- When starting a new web design project from scratch.
- When auditing a website that feels 'unprofessional' or 'off'.
- When optimizing a site for better user engagement or conversion rates.
- When establishing a design system or style guide for a web project.

## Anti-Patterns

- Don't use the same overused free fonts that make designs blend in with everyone else.
- Avoid eyeballing font sizes, line heights, or spacing without a mathematical system.
- Don't use more than two or three different colors in a single palette.
- Avoid designing for aesthetics alone without a clear conversion goal in mind.
- Don't build everything from scratch if a code snippet or tool can get you 80% of the way there.

## Workflow

1. Select unique fonts from sources like Fontshare and set up a type scale system at typescale.net.
2. Set up a 12-column layout grid and an 8-point spacing grid in Figma or your design tool.
3. Establish a limited color palette using the 60-30-10 rule and verify contrast ratios.
4. Apply visual hierarchy using proximity, size, contrast, and alignment to guide the scanner's eye.
5. Implement the design using basic HTML/CSS/JS, utilizing existing snippets for complex interactions.
6. Insert a single primary goal per page and place CTAs strategically throughout the layout.
7. Add trust signals like testimonials and social proof to reinforce the conversion goal.

## Examples

- Using the Major Third Scale to increase heading sizes by 25% from a 16px base.
- Using Chrome DevTools' CSS Overview to extract and analyze a professional site's color and type settings.
- Placing a 'Sign Up' CTA in the hero section and repeating it every few scrolls to maintain visibility.
